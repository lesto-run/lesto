# Edge, Deploy & Sites — v1 plan

Derived from `docs/reviews/edge-deploy.md`, reconciled with `docs/ROADMAP-V1.md` (which rules).
Packages: `@keel/cloudflare`, `@keel/deploy`, `@keel/sites` (+ the estate worker, the deployed
launch artifact). The prior "CF adapter bypasses all hardening" P0 is **fixed and test-locked**
— referenced as done. This plan **owns launch blocker #1** (the edge auth fence); auth-security
references it. Roadmap ruling (§6): the blessed v1 Cloudflare deploy stays `wrangler deploy`;
one remote `ReleaseStore` ships so release/rollback is real, but the Bet II uploader is post-1.0.

**The bar, every increment:** TS/ESM/Bun; oxlint/oxfmt clean; 100% vitest coverage on touched
packages; `bun run ws:typecheck` + the serial coverage gate green; coded errors; truthful doc
comments; one conventional commit on `main`.

## Increments (ordered)

1. **Fail closed on the edge secret** — `[Wave 0 | P0 | blocker #1, part 1]`
   Files: `examples/estate/src/edge.ts:238-240` — throw at first request when `env.SESSION_SECRET` is absent; the committed literal survives only under an explicit `KEEL_DEMO=1` binding. Document this as the framework's pattern for every secret-bearing Worker.
   Acceptance: an e2e proving an unset secret refuses to serve; demo mode requires the explicit binding; the secret-strength guard (auth-security item 1) applies in the same pass.

2. **Bring the edge twin to node-twin posture** — `[Wave 0 | P0 | blocker #1, part 2]`
   Files: `examples/estate/src/edge.ts` — mount `fromRequestMiddleware(secureStack({ originCheck: {} }))` + per-isolate rate limiting in `buildEdgeApp`; replace the passwordless `?as=` sign-in with `@keel/identity` (already shipped) or hard-gate it behind the demo binding; delete the duplicated `readCookie`/`SESSION_COOKIE` in favor of `@keel/identity`'s cookie module.
   Acceptance: a cross-origin POST is refused on the edge (e2e); `?as=` unreachable without the demo flag; sign-in throttled.

3. **Plumb `ExecutionContext` + `waitUntil` tracing** — `[Wave 4 | P1]` (the edge half of blocker #11; operability-dx item 3 owns the tracer/env contract)
   Files: `packages/cloudflare/src/fetch-handler.ts` — accept `(request, ctx?)`; flush the exporter via `ctx.waitUntil`; `examples/estate/worker.ts` wires the `OtlpHttpExporter` + structured JSON access log as the canonical OTLP-on-Workers reference.
   Acceptance: spans from a Worker request land in a local collector in the integration suite; no span loss after `return`.

4. **Ship seam to bytes** — `[Wave 3 | P1]`
   Files: `packages/deploy/src/ship.ts` (`read`/`put` → `Uint8Array`, `string` convenience overload), `packages/sites` `OutputSink` with it; widen `RenderResponse` body or fail with a coded error for non-string routes in static zones.
   Acceptance: a PNG/woff2 round-trips bit-exact through `shipStatic`; the latent corruption class is closed before any non-HTML asset flows.

5. **One real remote `ReleaseStore` (R2 / Workers Assets)** — `[Wave 3 | P1]`
   Files: new store in `packages/deploy` implementing `ReleaseStore` over fetch + SigV4 (share the core with data-persistence item 7's S3 backend — one signing implementation, two consumers); stage → health-gate → atomic pointer flip semantics preserved.
   Acceptance: `keel deploy --release` + `rollback` against an R2-shaped fake (and a recorded live smoke); the carefully-ordered release machinery is reachable on the headline target.

6. **Make `wranglerConfig` honest** — `[Wave 5 | P1]`
   Files: `packages/cloudflare/src/wrangler.ts` — grow `WranglerOptions` to cover what the real deploy needs (`alias`, `vars`, placement); add a round-trip test that regenerates `examples/estate/wrangler.jsonc` byte-identically, so drift (a compatibility_date bump, a binding rename) fails CI.
   Acceptance: the "Generated by `wranglerConfig`" claim in the committed file is true again.

7. **Edge polish batch** — `[Wave 5 | P2]` (one PR)
   Async Web-Crypto SHA-256 ETag/304 for buffered dynamic edge responses; structured JSON default edge access log (mirrors core-runtime item 4); optional `timeoutMs` racing dispatch with the coded 503; update the package doc example that imports node-flavored `dispatchSites` into a Worker to the pattern estate actually deploys.

## Owned elsewhere (do not duplicate)

- `secureStack`/identity internals the edge mounts → **auth-security**.
- The `Set-Cookie` multimap seam change in the fetch adapter → **core-runtime** item 5 (single cross-seam commit).
- Hashed island bundles + chunk generations → **ui-client** items 2/4.
- Tracer construction/env knobs → **operability-dx** item 3.

## Deferred post-1.0 (deliberate)

- A `worker`/`edge` target kind in the deploy plan model + routing-manifest-driven `withAssets` (today's assets-first probe is a documented portability shim) — lands with the Bet II assets substrate.
- `PublishFile` cache-policy field + `Cache-Control: immutable` story — same wave as content-hashed bundle filenames (Bet II).
- Release pruning (`prune(keep: n)` refusing `current`).
- The full incremental S3-compatible uploader + CDN invalidation (`keel deploy` Bet II headline) — post-1.0; v1's deploy story is `wrangler deploy` (edge) + `shipRelease` over R2/local (node/static).
