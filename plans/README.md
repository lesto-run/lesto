## Dependency notes

- **Run 009 first.** It de-risks every other plan's "full local gate" step by
  making `gate` / `gate:full` real (today's docs claim a CI parity that does not
  hold — the documented 0.1.7 near-miss class).
- **The `bun.lock` chain — run these SERIALLY in one worktree, in this order:**
  **002 → 012 → 017 → 016 → 007**, then **022**. All of them regenerate or edit
  `bun.lock`, and the Studio daemon commits to `main` concurrently (shared-worktree
  trap), so parallel lock regens will conflict. Rationale for the order: 002 is
  the urgent security bump; 012/017 are mechanical manifest edits that settle the
  dependency state; 016 is now small; 007 (L, staged, breaking ctor change) lands
  last so the big refactor isn't rebased over manifest churn. 022 uses a plain
  `bun install` (no full regen) so it is the lightest.
- **010 before 019** — 019 touches the same server.ts / fetch-handler.ts regions.
- **006 must land before the launch release cut (0.1.8)** — READMEs only reach npm
  at a cut (`scripts/lib/build-public.mjs` stages them), so landing it after the
  cut leaves the npm pages blank through the exact window it exists to fix.
- Everything else is disjoint by package and can run in parallel.

## Review outcome (7-agent fleet, 2026-07-11)

The review caught real defects in the plans. The most severe, all now fixed:

- **007 — false premise (BLOCKER).** `statusForError` (`web/src/harden.ts:198`) is
  a **closed code allowlist**, so branding `DocksError` alone can never map a
  content error to a 4xx. The original Stage-1 acceptance test was unsatisfiable
  in scope and could have been "passed" vacuously. Reframed to test
  `isLestoError`/`hasCode` brand recognition; the 4xx claim is dropped.
- **015 — the "fix" could crash the process.** `removeAllListeners()` on rebind
  strips the `'error'` handler from the still-live underlying socket → an uncaught
  exception on the mail path. Changed to targeted `off("data"/"error")`, plus a
  crash-regression test.
- **001 — wrong exception name.** `AbortSignal.timeout()` rejects with
  `TimeoutError` (default fetch/undici/workerd), not `AbortError` (Node http);
  keying on the name would miss the real production timeout. Now maps
  structurally, with a distinct `WEBHOOK_DELIVERY_TIMEOUT` code. The original
  test stub also deadlocked.
- **003 — the rewrite broke typecheck.** A boolean gate doesn't narrow
  `rehashed: string | undefined` for `describeHashCost`. Simplified to reuse the
  existing narrowing.
- **011 — wouldn't typecheck.** Header values are `HeaderMap = Record<string,
  string | string[]>`, not plain strings; and the policy `Vary` is itself
  multi-token.
- **021 — the one-liner bricks the watcher.** A rejected `pending` promise skips
  every subsequent reload; a terminal `.catch` is required.
- **009 — incomplete enumeration** would have recreated the false CI parity it
  exists to kill (dropped the four Postgres-service jobs and others).
- **006 — targeted the wrong packages.** The 13 headline batteries already have
  READMEs; the 29 missing ones are the core infra.
- **016 / 018 — premises were wrong.** `@content-collections/*` are dead,
  imported-nowhere devDeps (→ delete in 017, don't migrate); and the "7 duplicated
  sleeps" are injected defaults working as designed (only `content-mcp`'s are not).
  Both plans were substantially re-scoped and shrunk.
- **019 — only 1 of 4 "twins" is real.** Headers/body/target-guard are deliberate
  transport divergences; the body-reader parity claim was factually false. De-scoped
  to `parseQuery` + a conformance test.
- **026 — overstated.** Both ADR 0037 and 0039 are **Proposed**, not ratified, so
  `lesto add button` failing is unimplemented-feature state, not a broken promise.
  Reframed as forward design; de-prioritized.
- **022 — ADR 0045 rationale was inverted** (members CAN phantom-import root deps —
  which makes removal *more* valuable), and the `rm -f bun.lock` regen was unwarranted.

## Open decision the plans do NOT own

- **DIR-03 capstone**: no plan owns the finish line for "bring `content-*` to the
  house bar" — i.e. either removing the blanket `content-` prefix skip at

  *real* coverage exemption — it is NOT ADR 0035's mechanism) once the stack meets
  the bar, or ratifying its permanence in an ADR amendment. Until that is decided,
  "100% coverage" carries a standing asterisk. File it as an explicit decision.

## Findings considered and rejected (so they aren't re-audited)

- **god-file splits** — `cli/src/run.ts`, `runtime/src/server.ts`, `web/src/lesto.ts`
  are large but well-decomposed; only `runDev`/`handleAdmitted` are genuinely long.
  Split opportunistically when next touching them, not as a standalone refactor.
- **vacuous tests** — spot-checked the highest-risk negative assertions (queue DDL,
  identity secret-leak, mail SMTP, soft-nav); all load-bearing and corroborated by
  positive assertions / a real-socket baseline. No snapshot tests exist.
- **a shared cross-package `sleep`/`retry` package** — rejected: the copies are
  injected defaults (the testability convention working as designed), the queue
  already slices its drain for `stop()` responsiveness, and there is no cycle-free
  home. Only `content-mcp`'s non-injected sleeps were a real gap (plan 018).
- **merging the node/edge header, body, and request-target readers** — rejected:
  they are deliberate transport divergences with different error contracts; a
  forced merge would change one tier's behavior on a security boundary.
- **`identity.login()` max-password-length before the KDF** — body capped at 1 MiB,
  KDF cost dominated by its own parameters, per-account rate-limited. No amplification.
- **webhook SSRF deny-list doc/benchmark ranges** (`192.0.2.0/24`, NAT64) — not real
  SSRF targets; the metadata/loopback/RFC1918/link-local set is complete.
- **pre-commit hooks / `.env.example`** — deliberate: the Studio daemon's `update-ref`
  commits bypass git hooks; examples degrade gracefully without env vars.
- **better-sqlite3 11→12** — no EOL/prebuild cost today.
